Astro es un **marco de trabajo web todo-en-uno** para construir sitios web **rápidos y centrados en el contenido**.

¿Por qué elegir Astro en lugar de otro marco de trabajo web? Aquí tienes cinco principios de diseño fundamentales para ayudar a explicar por qué construimos Astro, los problemas que está diseñado para resolver y por qué Astro podría ser la mejor elección para tu proyecto o equipo.

#### Astro es...

1. [Centrado en el contenido](#centrado-en-el-contenido): Astro fue diseñado para sitios web ricos en contenido.
2. [Enfoque en el servidor](#enfoque-en-el-servidor): Los sitios web son más rápidos cuando generan HTML en el servidor.
3. [Rápido por defecto](#rapido-por-defecto): Debería ser imposible construir un sitio web lento en Astro.
4. [Fácil de usar](#facil-de-usar): No necesitas ser un experto para construir algo con Astro.
5. [Completo, pero flexible](#completo-pero-flexible): Más de 100 integraciones de Astro para elegir.

## centrado-en-el-contenido

**Astro fue diseñado para construir sitios web ricos en contenido.** Esto incluye la mayoría de los sitios de marketing, sitios de publicación, sitios de documentación, blogs, portafolios y algunos sitios de comercio electrónico.

Por otro lado, la mayoría de los marcos de trabajo web modernos están diseñados para construir *aplicaciones web*. Estos marcos funcionan mejor para crear experiencias más complejas, similares a aplicaciones en el navegador: paneles de administración para usuarios registrados, bandejas de entrada, redes sociales, listas de tareas e incluso aplicaciones similares a aplicaciones nativas como [Figma](https://figma.com/) y [Ping](https://ping.gg/).

Esta es una de las diferencias más importantes que debes entender acerca de Astro. El enfoque único de Astro en el contenido le permite tomar decisiones y ofrecer características de rendimiento incomparables que no tendrían sentido para marcos de trabajo web más centrados en aplicaciones.

:::tip
Si tu proyecto pertenece al segundo grupo de "aplicaciones", Astro podría no ser la elección adecuada para tu proyecto... ¡y eso está bien! Puedes considerar [Next.js](https://nextjs.org/) como una alternativa más centrada en aplicaciones a Astro.
:::

## enfoque-en-el-servidor

**Astro aprovecha la representación en el lado del servidor tanto como sea posible.** Este es el mismo enfoque que los marcos de trabajo tradicionales en el lado del servidor, como PHP, WordPress, Laravel, Ruby on Rails, etc., han estado utilizando durante décadas. Pero no necesitas aprender un segundo lenguaje en el lado del servidor para utilizarlo. Con Astro, todo sigue siendo solo HTML, CSS y JavaScript (o TypeScript, si lo prefieres).

Este enfoque contrasta con otros marcos web modernos en JavaScript como Next.js, SvelteKit, Nuxt, Remix y otros. Estos marcos requieren la representación en el lado del cliente de todo tu sitio web e incluyen la representación en el lado del servidor principalmente para abordar problemas de rendimiento. Este enfoque se ha denominado el **modelo de Aplicación de Página Única (SPA)**, en contraste con el enfoque **Aplicación de Múltiples Páginas (MPA)** de Astro.

El modelo SPA tiene sus beneficios. Sin embargo, estos vienen con la complejidad adicional y compensaciones en el rendimiento. Estas compensaciones afectan al rendimiento de las páginas, incluyendo métricas críticas como [Tiempo para Interacción (TTI)](https://web.dev/interactive/), lo cual no tiene mucho sentido para sitios web centrados en el contenido donde el rendimiento en la carga inicial es esencial.

## rapido-por-defecto

Un buen rendimiento siempre es importante, pero es *especialmente* crítico para sitios web centrados en el contenido. Se ha demostrado que un mal rendimiento provoca una pérdida de participación, conversiones y dinero. Por ejemplo:

- Cada 100 ms más rápido → 1% más de conversiones ([Mobify](https://web.dev/why-speed-matters/), ganando +$380,000 al año)
- 50% más rápido → 12% más de ventas ([AutoAnything](https://www.digitalcommerce360.com/2010/08/19/web-accelerator-revs-conversion-and-sales-autoanything/))
- 20% más rápido → 10% más de conversiones ([Furniture Village](https://www.thinkwithgoogle.com/intl/en-gb/marketing-strategies/app-and-mobile/furniture-village-and-greenlight-slash-page-load-times-boosting-user-experience/))
- 40% más rápido → 15% más de registros ([Pinterest](https://medium.com/pinterest-engineering/driving-user-growth-with-performance-improvements-cfc50dafadd7))
- 850 ms más rápido → 7% más de conversiones ([COOK](https://web.dev/why-speed-matters/))
- Cada 1 segundo más lento → 10% menos de usuarios ([BBC](https://www.creativebloq.com/features/how-the-bbc-builds-websites-that-scale))

En muchos marcos web, es fácil construir un sitio web que se ve genial durante el desarrollo pero que carga de manera dolorosamente lenta una vez desplegado. A menudo, JavaScript es el culpable, ya que los teléfonos de los usuarios y los dispositivos de menor potencia rara vez coinciden con la velocidad de la computadora portátil de un desarrollador.

La magia de Astro radica en cómo combina los dos valores mencionados anteriormente: un enfoque en el contenido con una arquitectura MPA en el lado del servidor, para tomar decisiones y ofrecer características que otros marcos no pueden. El resultado es un rendimiento web sorprendente para cada sitio web, directamente desde la caja. Nuestro objetivo: **Debería ser casi imposible construir un sitio web lento con Astro.**

Un sitio web de Astro puede [cargar un 40% más rápido con un 90% menos de JavaScript](https://twitter.com/t3dotgg/status/1437195415439360003) que el mismo sitio construido con el marco web React más popular. Pero no nos creas a nosotros, mira cómo el rendimiento de Astro deja sin palabras a Ryan Carniato (creador de Solid.js y Marko) [en este video](https://youtu.be/2ZEMb_H-LYE?t=8163).


## facil-de-usar

**El objetivo de Astro es ser accesible para todos los desarrolladores web.** Astro fue diseñado para sentirse familiar y asequible independientemente del nivel de habilidad o experiencia previa en desarrollo web.

Comenzamos asegurándonos de que puedas usar cualquier lenguaje de componentes de interfaz de usuario que ya conozcas. React, Preact, Svelte, Vue, Solid, Lit y varios otros son compatibles para crear nuevos componentes de interfaz de usuario en un proyecto de Astro.

También quisimos asegurarnos de que Astro tuviera un excelente lenguaje de componentes integrado. Para lograrlo, creamos nuestro propio lenguaje de interfaz de usuario `.astro`. Está fuertemente influenciado por HTML: ¡cualquier fragmento válido de HTML ya es un componente válido de Astro! Pero también combina algunas de nuestras características favoritas tomadas de otros lenguajes de componentes, como las expresiones JSX (React) y el enfoque en el alcance de CSS por defecto (Svelte y Vue). Esta cercanía con HTML también facilita el uso de la mejora progresiva y patrones de accesibilidad comunes sin ningún costo adicional.

Astro fue diseñado para ser menos complejo que otros marcos y lenguajes de interfaz de usuario. Una gran razón para esto es que Astro fue diseñado para representarse en el servidor, no en el navegador. Esto significa que no necesitas preocuparte por: hooks (React), cierres obsoletos (también React), refs (Vue), observables (Svelte), átomos, selectores, reacciones o derivaciones. No hay reactividad en el servidor, por lo que toda esa complejidad desaparece.

Una de nuestras frases favoritas es: **optar por la complejidad**. Diseñamos Astro para eliminar tanta "complejidad requerida" como sea posible de la experiencia del desarrollador, especialmente cuando te embarcas por primera vez. Puedes construir un ejemplo de sitio web "Hola Mundo" en Astro solo con HTML y CSS. Luego, cuando necesites construir algo más potente, puedes alcanzar gradualmente nuevas características y APIs a medida que avanzas.

## completo-pero-flexible

**Astro es un marco de trabajo web todo en uno que incluye todo lo que necesitas para construir un sitio web.** Astro incluye una sintaxis de componentes, enrutamiento basado en archivos, manejo de activos, un proceso de construcción, empaquetado, optimizaciones, obtención de datos y más. Puedes construir sitios web excepcionales sin necesidad de buscar más allá de las características principales de Astro.

Si necesitas más control, puedes extender Astro con más de [100+ integraciones](https://astro.build/integrations/) como [React](https://www.npmjs.com/package/@astrojs/react), [Svelte](https://www.npmjs.com/package/@astrojs/svelte), [Vue](https://www.npmjs.com/package/@astrojs/vue), [Tailwind CSS](https://www.npmjs.com/package/@astrojs/tailwind), [MDX](https://www.npmjs.com/package/@astrojs/mdx) y más. [Conecta tu CMS favorito](/es/guides/cms/) o [despliega en tu host preferido](/es/guides/deploy/) con un solo comando.

Astro es agnóstico en cuanto a la interfaz de usuario, lo que significa que puedes **Traer Tu Propio Marco de Interfaz de Usuario (TYPIU)**. React, Preact, Solid, Svelte, Vue y Lit son compatibles oficialmente en Astro. Incluso puedes combinar diferentes marcos en la misma página, facilitando las futuras migraciones y evitando quedar atrapado en un solo marco de trabajo.
